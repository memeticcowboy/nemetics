## Purpose

This directory holds **Insights**—pattern recognitions that emerge during reading, conversation, and framework navigation.

Not raw observations. Not conclusions. Just... **twists that haven't decided their fate yet**.

---

## Filing Principles

### 1. Every Insight Gets Context

Before an Insight is filed, it must carry:
- **Source:** Where did this emerge?
- **Habitat affinity:** Which processing context does it belong to?
- **Pattern connection:** What existing framework does it touch?
- **Action status:** What would need to be true for this to become actionable?

### 2. Structure Reveals Itself

Do not impose categories prematurely. Let the Insights cluster, then name the clusters.

Current provisional structure:
- `by_habitat/` — Insights sorted by where they emerged
- `by_pattern/` — Insights sorted by what they recognized
- `by_action_status/` — Insights sorted by what they need
- `context_maps/` — Cross-connections, emergent category proposals

### 3. Context Before Action

The filing itself is diagnostic. Before asking "what should we do?" ask:
- What Habitat does this insight belong to?
- What ε-conditions would allow it to unfold?
- What would premature action freeze?
